# 🧠 Claude

Claude是Anthropic开发的AI助手系列，包含网页版、桌面应用和命令行工具。Claude Code是终端原生的Agentic编程工具，据Anthropic报告贡献了GitHub约4%的公开提交量，SWE-bench Verified达到80.9%。

> **核心能力：** 200K上下文窗口（Opus 4.6 支持1M beta）、Agent Teams多智能体协作、`/goal` 自主目标追踪、MCP协议集成、Skills渐进式加载、Hooks生命周期自动化、CLAUDE.md分层记忆系统。

## 官方应用

| 应用 | 简介 |
|------|------|
| [Claude.ai](https://claude.ai/){:target="_blank"} | 网页版AI助手 |
| [Claude桌面应用](https://claude.ai/download){:target="_blank"} | Mac/Windows桌面客户端，支持本地文件访问 |
| [Claude Code](https://docs.anthropic.com/en/docs/claude-code/overview){:target="_blank"} | 终端原生的Agentic编程工具，支持自主完成复杂开发任务 |

## 热门Skills

| Skill | 功能 |
|-------|------|
| `/review` | 代码审查，分析代码问题 |
| `/init` | 初始化项目配置，生成CLAUDE.md |
| `/verify` | 验证代码变更是否达到预期效果 |
| `/loop` | 定时循环执行任务，适用于轮询和周期性检查 |
| `/run` | 启动并驱动应用，验证变更是否生效 |
| `/goal` | 定义完成条件，跨轮次自主工作直到目标达成 |
| `/commit` | 自动生成提交信息并创建commit |
| `/pr` | 创建Pull Request |

## Hooks

Claude Code支持在工具调用前后自动执行Shell命令，实现生命周期自动化：

| Hook事件 | 用途 |
|----------|------|
| `PreToolUse` | 工具调用前拦截或校验（如禁止危险命令） |
| `PostToolUse` | 工具调用后触发动作（如自动格式化） |
| `Notification` | 任务完成时发送通知（如桌面提醒、消息推送） |
| `Stop` | Agent停止时执行清理或汇总 |

## Skills

|-------|------|

| `$skill-creator` | 脚手架创建新技能 |
| `$skill-installer` | 从GitHub/本地路径安装技能 |

# 🤖 hermes

hermes是NousResearch开源的AI Agent框架（174K+ Stars），独特之处在于**内置学习循环**——每10次工具调用迭代触发技能提醒（nudge），促使Agent自主创建和管理技能，Curator模块7天周期自动评估、合并、修剪技能。

> **核心能力：** 20+聊天平台、自学习技能生成、70+内置工具、多Provider支持、MCP集成、插件架构。

## 插件

| 插件 | 功能 |
|------|------|
| **ArkClaw** | 火山引擎消息渠道集成与任务调度 |
| **OpenClaw Bridge** | 双向消息路由，自动加载 `skill.yaml` |
| **MCP Protocol v1.3** | 非侵入式能力扩展（API/数据库/IoT） |

docker配置
```
services:
 hermes:
  image: nousresearch/hermes-agent:latest
  container_name: hermes
  restart: unless-stopped
  command: gateway run
  ports:
   - "8642:8642"
  environment:
   - PATH=/opt/hermes/.venv/bin:/usr/local/bin:/usr/bin:/bin
  stdin_open: true
  tty: true
  volumes:
   - ./.hermes:/opt/data
  deploy:
   resources:
    limits:
     memory: 4G
     cpus: "2.0"
```

进入配置选择LLM，聊天通道目前支持QQ、飞书等。

`hermes setup`

# 🔧 PI Agent

Pi是Mario Zechner（libGDX/RoboVM作者）打造的极简开源AI编程Agent工具包（TypeScript开发，58K+ Stars）。设计哲学："极度可扩展，不强迫工作流"——故意不加MCP、子Agent、权限弹窗等内置功能，全部通过扩展实现。

> **核心能力：** 四种运行模式（交互式TUI / 管道输出 / RPC进程 / SDK嵌入）、会话树形分支与分叉、25+ LLM Provider统一API、TypeScript扩展系统、Agent Skills标准支持、Pi Packages生态分发。

## 官方资源

| 资源 | 简介 |
|------|------|
| [Pi](https://github.com/earendil-works/pi){:target="_blank"} | GitHub仓库，含CLI、Agent运行时、多Provider API、TUI库 |
| [官方网站](https://pi.dev/){:target="_blank"} | Pi Agent 官网 |
| [Pi Agent Core](https://www.npmjs.com/package/@earendil-works/pi-agent-core){:target="_blank"} | npm包，Agent运行时核心 |

## Extensions 扩展

Pi通过TypeScript扩展模块注册自定义工具、命令、热键、UI组件和子Agent：

| 类别 | 代表性扩展 |
|------|-----------|
| **安全/沙箱** | `nono`（内核级能力沙箱）、`sandbox`（OS级隔离）、`permission-gate` |
| **子Agent编排** | Plan mode、并行分发、链式执行、subagent协调 |
| **搜索/网络** | Brave、Exa、SearXNG、Kagi、Jina fetch、GitHub search |
| **模型Provider** | OpenRouter（Kimi/Gemini/Grok）、Groq、Together |
| **集成** | MCP适配器、Discord/Telegram桥接、SSH远程、Bitwarden |
| **自动化** | `loop`（周期轮询）、`until`（条件迭代） |

> 安装：`pi install npm:@包名` 或 `pi install git:github.com/作者/仓库`

将扩展+技能+提示模板+主题打包为npm/git可分发包，社区通过 `pi install` 一键安装。

# 🦞 OpenClaw

OpenClaw是2026年最热门的开源自托管个人AI助手（37.6万+ Stars，MIT），定位"把LLM从对话式AI升级为行动式AI"。网关是控制平面，产品是助手本身。七层架构：渠道层→网关→Agent运行时→技能系统→记忆→插件→本地执行。

> **核心能力：** 20+聊天平台（WhatsApp/Telegram/Discord/微信/飞书/钉钉/Slack等）、200+ LLM支持、四层记忆（会话→日志→MEMORY.md→向量检索）、7×24自主运行（HEARTBEAT.md + Cron）、Commander-Worker多Agent编排、A2A通信、Docker沙箱安全执行。

## 官方资源

| 资源 | 简介 |
|------|------|
| [OpenClaw](https://github.com/openclaw/openclaw){:target="_blank"} | 开源自托管AI助手 |
| [awesome-openclaw](https://github.com/SamurAIGPT/awesome-openclaw){:target="_blank"} | 资源、技能、教程汇总 |
| [中文文档](https://github.com/liyupi/openclaw-guide){:target="_blank"} | OpenClaw中文指南，程序员鱼皮维护 |

## ClawHub 技能市场

ClawHub是OpenClaw内置的技能市场（13,729+技能，31个分类），支持SHA-256密码学验证：

```bash
openclaw skills search <关键词>    # 搜索技能
openclaw skills install <技能名>   # 安装技能
openclaw skills update             # 更新技能
```

> **跨生态兼容：** 可直接发现和导入来自Claude、Codex、Cursor平台的插件包，自动映射为OpenClaw格式。
>
> **安全隐患：** 约7.1%的技能存在API密钥泄露风险（Snyk报告），建议隔离容器运行并审计技能代码。

## 插件

OpenClaw插件架构分为四类：Channel / Memory / Tool / Provider：

| 插件/特性 | 功能 |
|-----------|------|
| **Plugin SDK** | `openclaw/plugin-sdk/*` 子路径，窄接口策略保证版本兼容 |
| **Exa / Tavily / Firecrawl** | 内置搜索引擎插件，无需额外API配置 |
| `/btw` | 侧边提问命令，不污染主对话上下文 |
| **Multi-browser** | 支持Brave、Edge等浏览器自动化 |
